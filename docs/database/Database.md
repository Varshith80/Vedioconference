# Database

> Source of truth for the data layer. Anything that contradicts this
> document is a bug in the code.

## 1. Conventions

- All primary keys are `uuid` generated by `gen_random_uuid()`.
- Timestamps are `timestamptz` (UTC). The application is responsible
  for rendering in the user's locale.
- Money is stored as **integer cents** (`integer` column).
- Soft delete is **not** used; we hard-delete where required
  (resources, audit logs are append-only).
- Every mutable table has `created_at` and `updated_at` columns with a
  `BEFORE UPDATE` trigger that bumps `updated_at`.

## 3. Tables

| Table              | Purpose | Key columns |
|--------------------|---------|-------------|
| `auth.users`       | Supabase-managed users | `id` |
| `profiles`         | Public user info + role | `id`, `email`, `role` |
| `tutors`           | Tutor-specific data | `profile_id`, `calendly_event_uri`, `zoom_user_id`, `hourly_rate` |
| `courses`          | Tutoring offerings | `slug`, `subject`, `level`, `price_cents`, `is_published` |
| `course_tutors`    | Course ↔ tutor M:N | `course_id`, `tutor_id` |
| `bookings`         | Scheduled classes | `student_id`, `tutor_id`, `course_id`, `status`, `scheduled_start` |
| `payments`         | Stripe payments per booking | `booking_id`, `stripe_payment_intent_id`, `status`, `amount_cents` |
| `meeting_links`    | Zoom meeting per booking | `booking_id` (UNIQUE), `join_url`, `start_url` |
| `resources`        | Study material files | `course_id`, `tutor_id`, `file_path`, `visibility` |
| `resource_grants`  | Per-booking access to a resource | `(resource_id, booking_id)` |
| `notifications`    | In-app + sent-email mirror | `user_id`, `type`, `channel`, `read_at` |
| `audit_logs`       | Append-only log of mutations | `table_name`, `row_id`, `action`, `changes` |

## 4. Enums

- `user_role`        : `student | admin | super_admin`
- `booking_status`   : `pending_payment | confirmed | completed | cancelled | no_show | rescheduled`
- `payment_status`   : `pending | succeeded | failed | refunded | partially_refunded`
- `payment_provider` : `stripe | other`

## 5. Indexes

All FK columns and the columns used in dashboard filters are indexed:

- `bookings(student_id)`, `bookings(tutor_id)`, `bookings(course_id)`,
  `bookings(status)`, `bookings(scheduled_start)`,
  `bookings(stripe_session_id)`, `bookings(calendly_event_uri)`
- `payments(booking_id)`, `payments(status)`
- `meeting_links(booking_id)` (UNIQUE)
- `resources(course_id)`, `resources(tutor_id)`, `resources(visibility)`
- `notifications(user_id)`, `notifications(type)`, `notifications(read_at)`
- `audit_logs(table_name)`, `audit_logs(row_id)`, `audit_logs(actor_id)`,
  `audit_logs(created_at DESC)`

## 6. Row-Level Security

- `profiles` — owner-or-admin read; owner self-insert on signup; only
  super admin may delete.
- `tutors` — public read of `is_published = true`; admin write.
- `courses` — public read of `is_published = true`; admin write.
- `course_tutors` — public read; admin write.
- `bookings` — student can read/write their own; admin can do anything.
- `payments` — student can read payments of their own bookings; admin
  write only (writes come from n8n via service role).
- `meeting_links` — same visibility as `bookings`.
- `resources` — public or `enrolled` (joined via `resource_grants`) or
  `private` (only the uploader).
- `notifications` — owner read, owner update, admin/owner insert.
- `audit_logs` — admin read only; inserts come from triggers.

## 7. Storage

| Bucket          | Public | Write policy |
|-----------------|--------|--------------|
| `avatars`       | yes    | `auth.uid()` is the first path segment |
| `course-covers` | yes    | admin only |
| `resources`     | no     | admin or uploader |

Signed URLs are issued for the `resources` bucket (TTL 1 hour).

## 8. Migrations

Numbered and idempotent:

```
supabase/migrations/
├── 20260707000001_extensions_and_helpers.sql
├── 20260707000002_profiles_and_roles.sql
├── 20260707000003_tutors_courses.sql
├── 20260707000004_bookings_payments.sql
├── 20260707000005_resources_notifications_audit.sql
├── 20260707000006_rls_policies.sql
└── 20260707000007_storage_buckets.sql
```

Run them with the Supabase CLI:

```bash
supabase db push           # apply to linked project
supabase db reset          # drop + migrate + seed (dev only)
```

## 9. Seed data

`supabase/seed/000_seed.sql` creates:

- One admin (`admin@example.com` / `Admin#1234`)
- One demo student (`student@example.com` / `Student#1234`)
- Three published courses
- One published tutor

**Do not** run seed in production.

## 10. Type generation

```bash
pnpm db:types    # writes apps/web/types/database.generated.ts
```

The generated file is consumed by `services/`, `app/api/`, and
`lib/supabase/*` to provide end-to-end type safety.
